19 /* The I2CDevice class is intended as an abstract class form which all i2c
20  * devices will be derived.
21  * It provides per device i2c fucntionality as opposed to per bus making code
22  * and device managment much simpler. Each device specifically has it's own
23  * timeout value in milliseconds, i2c bus address, i2c bus reference, and error
24  * history state variables.
25  * The class also wraps common i2c_t3 wire function calls to manage the device
26  * device specific variables (error tracking, device specific timeouts, etc.).
27  * See https://github.com/nox771/i2c_t3 for more information about the teensy
28  * i2c library.
29  * The error history state variables operate in the following manner. Each
30  * device has two variables tracking errors. One tracks recent history and is a
31  * boolean that holds true if an error has occurred since the last call to
32  * i2c_pop_errors() and holds false otherwise. The second variable counts
33  * consecutive communication failures with the device and is updated with a call
34  * to i2c_pop_errors(). i2c_pop_errors should therefore only be called once
35  * during a communication with an i2c device (this includes configuration,
36  * requesting, and recieving that is needed to obtain data). Otherwise,
37  * i2c_peek_errors() can be used to check the recent history of i2c errors.
38  * For consistency, all member functions of this class are prefixed with i2c_.
39  */
